Overview

What is a Dark Factory?

A lights-out, fully automated manufacturing facility that operates without human presence — completely dependent on reliable machines and intelligent maintenance systems.

🏭

Dark Factory Concept

Dark factories (lights-out factories) operate in complete darkness without human workers on the floor. All processes are automated, making machine health monitoring and predictive maintenance absolutely critical — any unexpected breakdown causes significant operational loss.

🔮

Predictive Maintenance

Unlike reactive (fix-after-break) or scheduled maintenance, predictive maintenance uses real-time sensor data and machine learning models to forecast when a component will fail, enabling maintenance to be scheduled at the optimal time — minimizing cost and downtime.

🧠

Our ML Approach

We combine a physics-of-failure theoretical model with a k-Nearest Neighbours (kNN, k=3) regressor. Four sensor readings (vibration, temperature, pressure, operating hours) are MinMax-scaled and fed — together with LabelEncoded component type — into a distance-weighted kNN to predict the Remaining Useful Life (RUL) in hours.

📊

Open-Source Data

The dataset is sourced from Kaggle and contains 1,000 records of industrial construction machine components — hydraulic cylinders, gears, and engines — each described by 5 key sensor parameters, with annotated RUL values ranging from 51 to 999 hours.

🧬 Health Index Construction

Sensor Weights (W):
  Vibration      → 0.35 (most critical)
  Temperature    → 0.25
  Pressure       → 0.20
  Operating Hours→ 0.20

Normalize: MinMaxScaler (fitted on 1,000 rows)
norm = scaler.transform(sensors)
deg  = (clip(norm,0)^1.3 · W) × 2.0
HI   = exp(−deg)

Status:
  HI > 0.60     → 🟢 Healthy
  HI 0.35–0.60  → 🟡 Warning
  HI < 0.35     → 🔴 Critical

⚗️ Physics-Based Target Blending

RUL_Physics = 50 + (1000−50) × HI^1.4

Concave decay (α=1.4):
  • Amplifies degradation near end-of-life
  • Realistic wear-out curve

Blended Target (80/20):
  RUL_target = 0.80 × RUL_Physics
             + 0.20 × RUL_Dataset

→ Reduces noise while preserving
  physics-based domain knowledge

Mathematics

Theoretical RUL Formula

Physics-of-Failure / Stress-Life model — a first-principles approach providing interpretable, academically defensible RUL estimates independent of the ML black-box.

Step 1

Normalize Inputs (0 → 1)

V_norm = Vibration / 5.0
T_norm = (Temperature − 40) / 60
P_norm = (Pressure − 50) / 250
H_norm = Operating_Hours / 5000
Step 2

Compute Degradation Index (Component-Specific)

DI = 1.5·V² + 0.8·T1.5 + 1.2·P1.2 + 0.7·H
DI_max = 4.2  |  L_max = 1,200 hrs
DI = 1.8·V² + 1.0·T1.5 + 0.6·P1.2 + 1.0·H
DI_max = 4.4  |  L_max = 1,100 hrs
DI = 1.0·V² + 1.5·T1.5 + 0.9·P1.2 + 0.8·H
DI_max = 4.2  |  L_max = 1,400 hrs
Step 3

Normalize Degradation Index

DI_norm = DI / DI_max     ∈ [0, 1]
Step 4

Exponential Decay Model (Miner's Rule analogy)

RUL_raw = L_max × e−2.5 × DI_norm
Mimics Arrhenius / Coffin-Manson fatigue life models
Step 5

Scale to Dataset Range [51, 999 hrs]

RUL_scaled = 51 + (RUL_raw − L_max·e−2.5)
               ÷ (L_max − L_max·e−2.5) × 948
Step 6

Bound & Round

RUL_final = max(51, min(999, round(RUL_scaled)))

Why kNN for This Dataset?

Rationale

The dataset's RUL column has very low correlation with sensor readings (<0.04). This means parametric models (Linear Regression, Random Forest) cannot generalise — they would require memorisation. kNN with k=3 provides the best neighbourhood approximation: similar inputs → similar outputs.

Sensor↔RUL corr.< 0.04
RUL range51–999 h

📌 Key Findings

  • ▶ kNN (k=3) chosen because dataset RUL has near-zero correlation (<0.04) with sensors — neighbourhood lookup beats any parametric model
  • ▶ Distance weighting (1/d) ensures closer neighbours have stronger influence on the predicted RUL
  • ▶ MinMaxScaler fitted on all 1,000 rows ensures sensor ranges are properly normalised before distance computation
  • ▶ The exponential RUL formula provides an independent, physics-grounded baseline for comparison
  • ▶ Gear components are most sensitive to vibration; Engines to temperature — reflected in theoretical stress weights

🎓 Academic Contributions

  • ▶ First-principles physics-of-failure model provides interpretable theoretical RUL baseline independent of ML
  • ▶ Health Index (exp-decay of weighted normalised sensors) quantifies component degradation state
  • ▶ Component-specific stress weighting makes the formula generalizable to other industrial domains
  • ▶ Demonstrates that when labels are stochastic, instance-based (kNN) learning outperforms global parametric models
  • ▶ Maintenance alert thresholds (100/300/600 hrs) provide actionable industry-ready outputs
